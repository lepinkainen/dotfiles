# Analyze and plan a solution for GitHub issue: $ARGUMENTS

Follow these steps:

1. Use `gh issue view` to get the issue details
2. Understand the problem described in the issue
3. Search the codebase for relevant files
4. Think of a plan to implement the necessary changes to fix the issue
5. DO NOT IMPLEMENT THE FIX YET
6. Update the issue DESCRIPTION with your plan
7. Check what labels are available and add relevant ones to the issue
   - You can use `gh label list` to see available labels
   - You can create new labels if needed using `gh label create` if it is necessary

Remember to use the GitHub CLI (`gh`) for all GitHub-related tasks.
